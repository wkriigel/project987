# Filters Overview

Inputs used above the results table to narrow data. Filters should:

- Be generic and composable (avoid business logic in UI components).
- Use brand tokens via theme (`roles`).
- Be controlled values to integrate with app state.

## Current primitives

- `FilterSelect`: generic labeled single-select used for Generation.
